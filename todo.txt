# Fix: replace onSelect with onKey in every real-work delegate

## Why
`onTap` returning `true` does NOT stop taps from firing `onSelect` on the fr955.
Garmin's firmware translates a tap into `onSelect` before the app sees it as a touch
event — this is a confirmed platform behaviour and cannot be intercepted via `onTap`.

The only reliable fix: handle the START button via `onKey` (which is physically
button-only, never fired by a tap) and remove `onSelect` from every delegate that
does real work. Keep `onTap`/`onHold`/`onSwipe` returning `true` as they are.

`Menu2InputDelegate` subclasses (`IdleMenuDelegate`, `StopMenuDelegate`,
`CustomListDelegate`, `SuggestionChooserDelegate`) are exempt — Garmin's `Menu2`
drives their item selection internally and `onKey` doesn't route through them the
same way. Leave those as-is.

---

## 1. `HeatAccDelegate` — `source/HeatAccDelegate.mc` line ~34

REMOVE:
```monkeyc
function onSelect() {
    var state = _view.getState();
    if (state == Config.STATE_IDLE) {
        _view.startSession();
    } else if (state == Config.STATE_HEAT || state == Config.STATE_REST) {
        _view.pauseSession();
    }
    return true;
}
```

REPLACE WITH:
```monkeyc
function onKey(keyEvent as WatchUi.KeyEvent) as Lang.Boolean {
    var key = keyEvent.getKey();
    if (key == WatchUi.KEY_ENTER || key == WatchUi.KEY_START) {
        var state = _view.getState();
        if (state == Config.STATE_IDLE) {
            _view.startSession();
        } else if (state == Config.STATE_HEAT || state == Config.STATE_REST) {
            _view.pauseSession();
        }
        return true;
    }
    return false;
}
```

## 5. `PostSessionDelegate` — `source/SessionData.mc` line ~63

REMOVE:
```monkeyc
function onSelect() {
    var v = new HeatAccView();
    WatchUi.switchToView(v, new HeatAccDelegate(v), WatchUi.SLIDE_LEFT);
    return true;
}
```

REPLACE WITH:
```monkeyc
function onKey(keyEvent as WatchUi.KeyEvent) as Lang.Boolean {
    var key = keyEvent.getKey();
    if (key == WatchUi.KEY_ENTER || key == WatchUi.KEY_START) {
        var v = new HeatAccView();
        WatchUi.switchToView(v, new HeatAccDelegate(v), WatchUi.SLIDE_LEFT);
        return true;
    }
    // Keep existing KEY_UP / KEY_DOWN page navigation
    var k = keyEvent.getKey();
    if (k == WatchUi.KEY_DOWN) {
        var next = (_page + 1) % 5;
        WatchUi.switchToView(makeView(next),
            new PostSessionDelegate(next, _data), WatchUi.SLIDE_UP);
        return true;
    } else if (k == WatchUi.KEY_UP) {
        var prev = (_page + 4) % 5;
        WatchUi.switchToView(makeView(prev),
            new PostSessionDelegate(prev, _data), WatchUi.SLIDE_DOWN);
        return true;
    }
    return false;
}
```

NOTE: `PostSessionDelegate` already has an `onKey` for UP/DOWN paging — merge
the START check into that existing function rather than adding a second `onKey`.
The merged version should handle KEY_START/KEY_ENTER first, then KEY_DOWN/KEY_UP.

## Do NOT change
- `IdleMenuDelegate.onSelect(item)` — Menu2InputDelegate, Garmin handles internally
- `StopMenuDelegate.onSelect(item)` — same
- `CustomListDelegate.onSelect(item)` — same
- `SuggestionChooserDelegate.onSelect(item)` — same
- All `onTap`/`onHold`/`onSwipe` returning `true` — keep them

---

## Acceptance criteria
- Tapping anywhere on the idle screen does NOT start a session.
- Tapping anywhere during HEAT/REST does NOT open the pause menu.
- Tapping the suggestion benefit screen does NOT arm the workout.
- The physical START button still starts/pauses sessions and arms workouts normally.
- UP/DOWN physical buttons still page and scroll normally.
