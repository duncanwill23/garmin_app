## DESIGN UNDERSTANDING (read first)

- **Dose lever is TIME, not intensity.** Progress by extending heat time / adding
  rounds; the HR band stays 65–80% HRmax.
- **Three control loops:** progression (slow, by sessions) sets today's ceiling;
  readiness (acute) only scales DOWN; adaptation feedback (HR-at-load trend) decides
  advance / hold / maintenance.
- **Cold start:** first 5 sessions are a CALIBRATION window — fixed safe ramp, coarse
  readiness only, acclimation % withheld ("building baseline n/5"). Personalize after.
- **Acclimation is a PEAK you time to a race, not a level you hold** (it decays
  ~2.5%/day). So race logic is periodized and only matters near the race.
- **Idle is always FREE.** A workout is something the user deliberately loads (arms)
  and can deliberately unload (Cancel workout).
- **Two suggestions, user chooses:** Progression (always) and Race (only within the
  race window). Each is clearly labeled.
- **Don't overfit:** discrete rungs + a few readiness bands + evidence-based defaults
  as the prior; personal data only nudges within bounds; a gap regresses a rung.

## 2. Acute HR alerts (advisory only — never block)
`Config.mc`:
```
const HR_HIGH_PCT      = 0.92;
const HR_SUSTAINED_PCT = 0.85;
const HR_SUSTAINED_SEC = 12 * 60;
```
`HeatAccView` (during HEAT, using `_dose.pctHrMax(hr)`): momentary `>= HR_HIGH_PCT*100`
→ "HR very high" + one `Attention.vibrate` (re-arm after dropping below); track
`_sustainedHighSec` (increment `>= HR_SUSTAINED_PCT*100`, reset below), exceed
`HR_SUSTAINED_SEC` → "high HR a while — consider cooling down" + vibrate. Draw the
banner. `using Toybox.Attention;` guarded by `Attention has :vibrate`. Dose keeps
accruing; these warn only.

## 4. Readiness gathering  (`Readiness.mc`, all NULL-SAFE — often null in the sim)
- Body Battery — `SensorHistory.getBodyBatteryHistory()` newest (primary).
- Stress — `SensorHistory.getStressHistory()` newest.
- Today's activity — `ActivityMonitor.getInfo()` (intensity minutes, steps).
- Training Status string — Complications (best-effort; down-weight "overreaching").
- Recent heat session — `TrendStore.daysSinceLast()`.
- Manual "feeling run down?" per-day flag (also the null fallback).
Compute a factor [0,1] (Body Battery dominant; knocked down by stress, big activity,
overreaching, recent heat, manual flag) → band FULL/TRIM/SHORT/REST + reason string.
Readiness only scales DOWN. During calibration use only coarse signals.

## 5. Progression engine  (`WorkoutSuggester.mc`)
`suggestProgression(modality, readiness)` → `Workout(kind=PROGRESSION)` + rationale +
projected gain. Uses `TrendStore` (`getHistoryV().size()` = sessions, `daysIntoProgram`,
`inMaintenancePhase`, `daysSinceLast`, `estimatedDecayPct`, `currentAcclimation`).
Ladder (rungs by sessions; numbers in Config, tunable):
- Calibration (1–5): 1–2 rounds, ~8–12 min total.
- Build (~6–8, induction): 2–3 rounds → ~20–30 min.
- Full induction (~9+, `daysIntoProgram < INDUCTION_STABILIZE_DAYS`): ~3 rounds, ~30 min.
- Maintenance (`inMaintenancePhase()`): ~2 rounds, ~20 min.
Never exceed `SESSION_TARGET_SEC`; rests ~`REST_TARGET_SEC`; band 65–80% HRmax.
Modifiers: readiness band scales DOWN; gap (`daysSinceLast()>3`) regresses a rung +
shows "~X% decayed". Projected gain = simulate the leaky integrator for the dose
WITHOUT committing.

## 9. Set race date + race periodization
- Entry: on-watch date picker + an app-settings date field; store in
  `Application.Storage`. (Garmin's Primary Race date is NOT readable via the on-watch
  SDK — user enters it. Optionally probe Complications, don't depend on it.)
- `daysToRace = raceDate - today`.
- **Dormant outside the window:** if no race OR `daysToRace > RACE_WINDOW_DAYS`, do NOT
  generate a Race suggestion — only Progression runs. The Set-race-date screen shows
  the countdown and "race plan activates on [raceDate − RACE_WINDOW_DAYS]".
- **In window** (`0 < daysToRace <= RACE_WINDOW_DAYS`): `suggestRace(...)` →
  `Workout(kind=RACE)`:
  - `daysToRace > RACE_TAPER_DAYS` → BUILD: full induction ramp toward
    `RACE_TARGET_ACCL`, 3–5×/week (copy RACE_BUILD).
  - `daysToRace <= RACE_TAPER_DAYS` → TAPER: light maintenance, finish hard heat,
    1–2 short sessions (copy RACE_TAPER).
  - If too little time to reach target → add the RACE_INSUFFICIENT note.
- Sessions-needed: simulate the leaky integrator forward at a candidate cadence to hit
  `RACE_TARGET_ACCL` by race day; surface countdown, "~F×/week, ~N total", progress
  ("8 done — on track / behind by 2").

## 11. Explanation copy (Screen 2) — templated; `[..]` are fills
Show the kind badge above the copy: "Progression" or "Race prep · [D] days out".

- **CALIBRATION:** "Starter session — we're still learning how you respond, so keep it
  easy: [N] rounds of [H] min with [R] min rests. Stop early if you've had enough;
  building tolerance matters more than hitting the clock right now."
- **INDUCTION_BUILD:** "Building your acclimation. [N] × [H] min in the heat (~[T] min
  total) at [lo]–[hi] bpm, [R] min rests. The adaptation comes from consistency — aim
  for 3–5 sessions a week."
- **INDUCTION_FULL:** "Full induction session: [N] × [H] min (~[T] min total) at [lo]–
  [hi] bpm. This is the core of the block — most of your heart-rate and blood-volume
  adaptation is built here."
- **MAINTENANCE:** "Maintenance day. You've built the adaptation; now you're holding
  it. Lighter today — [N] × [H] min. Two to three a week keeps it from fading."
- **POST_GAP:** "Welcome back. It's been [D] days, so roughly [X]% of your acclimation
  has faded. Easing back in with [N] × [H] min and rebuilding from here."
- **READINESS_TRIM:** "Your body's a bit run down today ([reason], Body Battery [BB]),
  so this is scaled back to [N] × [H] min. A lighter day still counts."
- **READINESS_REST:** "Low readiness today ([reason], Body Battery [BB]). The honest
  call is rest — heat on a depleted body adds strain without much adaptation."
- **RACE_BUILD:** "Race prep — [D] days out. Building you to peak right at race day:
  [N] × [H] min (~[T] min total) at [lo]–[hi] bpm. Keep it consistent through the next
  couple of weeks."
- **RACE_TAPER:** "Race week — [D] days out. Keep heat light and finish hard heat about
  a week before race day. Short maintenance only today: [N] × [H] min."
- **RACE_INSUFFICIENT:** "Heads up: your race is [D] days away, and acclimation takes
  about two weeks to build. You'll still gain something, but there isn't time to fully
  acclimate — focus on consistency from here."

## 12. Auto-run a workout (the watch drives it)
`HeatAccView` gains `_workout`, `_stepIndex`, `_workoutMode`. `startSession(workout)`:
if non-null, load steps, `_stepIndex=0`, `_workoutMode=true`, start at step 0; if null,
today's free session unchanged. (Idle passes `_armedWorkout`.)
Adapted heat/rest screens (copy current ones, change the timer + add context):
- **Time counts DOWN** — remaining in the current block; show "Round N of M" + a
  one-line next-block preview.
- **Auto-advance:** in `onTick`, when `_phaseElapsed >= step.durationSec`, advance —
  reuse the existing HEAT↔REST transition from `markLap` (`_dose.startNewRound()` on a
  new heat block, `_session.addLap(...)`), fire `Attention.vibrate` + a cue ("Next:
  Rest 3:00").
- **LAP (bottom-right) = SKIP the current block** → next step (lap isn't needed inside
  a workout, so it's repurposed to skip).
- **START = pause** → existing `StopMenu` (Resume/Save/Discard), unchanged.
- **Over-run fallback:** after the LAST step, switch to FREE mode — the count-UP manual
  screen where LAP reverts to the heat↔rest toggle — so the user keeps going past the
  workout. Saving is the same.
- **Free session (no workout):** unchanged.
All modes: dose only in HEAT, recording through REST, per-round drift, HR recovery at
the 60-s rest mark.

## Acceptance criteria
- App always opens FREE; START runs a free session; no questionnaire.
- Hold-UP opens a menu (Suggestion / Custom / Set race date); "Cancel workout" appears
  only when a workout is armed and returns to free. Three dots hint at the menu.
- Suggestion tab offers Progression always, and Race only when the race is within
  `RACE_WINDOW_DAYS`; when both exist the user picks, and each is badged by kind.
- Each suggestion has 3 paged screens (bar chart, explanation, expected benefit);
  "Use this workout" arms it and returns to idle; START on idle begins it.
- A workout auto-advances with countdown timers, vibration cues, "round N of M"; LAP
  skips a block; START pauses; after the last block it falls into the free count-up
  screen; free mode unchanged.
- Outside the race window the race date is dormant (Progression only) and the
  Set-race-date screen shows the countdown + activation date.
- Suggestions ramp by sessions, scale DOWN with low readiness, regress after a gap;
  the Race suggestion builds then tapers and warns if there isn't time.
- First 5 sessions show "building baseline (n/5)" instead of an acclimation %.
- Custom workouts build on-watch and via phone app-settings slots, list, and arm.
- Steam shows "est." everywhere; HR-proxy line shows on live screens; manifest trimmed,
  Positioning gone.
- Readiness reads are null-safe (no crashes when Body Battery / status are unavailable).
