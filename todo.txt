# Fix: remove fake enableAlerts + soften same-day rest-day trigger

Two unrelated fixes.

---

## PART 1 — Vibration/sound: remove the misleading `:enableAlerts`

`:enableAlerts` is NOT a real `createSession` option in Garmin's ActivityRecording
API — it compiles but is silently ignored, so it does nothing. The tones/vibrations
heard at session start/stop and on each lap are the WATCH'S OWN activity alerts,
controlled by a device setting, not by app code. An app cannot suppress them.

### 1a. `source/SessionManager.mc` — drop the dead option
FIND:
```monkeyc
function start(modality, silent) {
    _session = ActivityRecording.createSession({
        :name         => "Heat Acclimation",
        :sport        => Config.REC_SPORT,
        :subSport     => Config.REC_SUB_SPORT,
        :enableAlerts => !silent
    });
```
REPLACE WITH:
```monkeyc
function start(modality) {
    _session = ActivityRecording.createSession({
        :name     => "Heat Acclimation",
        :sport    => Config.REC_SPORT,
        :subSport => Config.REC_SUB_SPORT
    });
```
(The `silent` param is no longer needed here.)

### 1b. `source/HeatAccView.mc` — revert the call
FIND:
```monkeyc
_session.start(_modality, _silent);
```
REPLACE WITH:
```monkeyc
_session.start(_modality);
```

### 1c. Keep the in-app silence toggle — it still mutes the APP'S OWN cues
The `_silent` flag and `vibrateOnce()` gating are still useful: they mute the app's
own `Attention.vibrate` calls (HR alerts, step-advance cues). Leave all of that as-is.
Only the fake `:enableAlerts` is being removed.

### 1d. Clarify the menu label so users know its scope
`resources/strings/strings.xml` — change:
```xml
<string id="MenuSilent">Silence alerts</string>
```
to:
```xml
<string id="MenuSilent">Mute app cues</string>
```
And add a new string for an on-screen note:
```xml
<string id="SystemTonesNote">Lap tones: watch Settings &gt; System &gt; Sounds</string>
```
Optionally display `SystemTonesNote` as a small footer line on the idle screen (or in
the menu) so the user knows the lap/start/stop tones are silenced via the watch's own
Sounds &amp; Vibration setting, not the app. (fr955: Settings &gt; System &gt; Sounds
and Vibration — turn off Tones/Vibration, or set per-activity.)

## PART 2 — Stop a same-day session from forcing a REST day

### The cause
In `Readiness.assess()`, BB 39 starts at SHORT (band 2). If a heat session was already
done today (`daysSinceLast() == 0`), the same-day knockdown bumps SHORT → REST (band 3),
producing a rest-day suggestion. The threshold itself is fine; it's the stacked
same-day knockdown that tips it over.

### The fix — cap the same-day-session knockdown at SHORT
The same-day session should never, on its own, push a user all the way to a full REST
day — it should at most land them at SHORT. Change that one knockdown so it can't
exceed SHORT.

`source/Readiness.mc` — FIND:
```monkeyc
// Same-day heat session: slight knockdown (< 24 h recovery)
var dsl = TrendStore.daysSinceLast();
if (dsl != null && dsl == 0) {
    band   = bandDown(band);
    reason = reason + " · session today";
}
```
REPLACE WITH:
```monkeyc
// Same-day heat session: nudge down, but never past SHORT on its own.
// (Doing a second session same-day adds strain, but shouldn't force a full
//  rest day unless Body Battery is already very low.)
var dsl = TrendStore.daysSinceLast();
if (dsl != null && dsl == 0) {
    var nudged = bandDown(band);
    band   = (nudged > Config.READINESS_SHORT) ? Config.READINESS_SHORT : nudged;
    reason = reason + " · session today";
}
```

### Result
- BB 39 + session today → SHORT (a short session), not REST.
- BB < 25 (very low) still yields REST directly from Body Battery, regardless.
- Genuine multi-factor depletion (e.g. low BB + high stress + heavy activity) can still
  reach REST through the other knockdowns — only the same-day-session one is capped.

---

## Acceptance criteria
- No `:enableAlerts` anywhere in the codebase; `start()` takes only `modality`.
- The app's own HR/step cues still respect the silence toggle.
- The menu/idle communicates that lap tones are a watch setting.
- With BB 39 and a session already completed today, "Suggestion" returns a SHORT
  workout, not a rest day.
- A rest day still appears for very low Body Battery (<25) or multiple stacked
  knockdowns.
