# Spec: arc-ring workout display (preview + live progress)

Replaces the bar-chart visual in the workout flow with a segmented ring around the
bezel. Refines `BUILD_TODO.md` §7 (suggestion screen 1), §8 (custom preview), and
§12 (auto-run screens). Keep rendering resolution-relative.

## Concept
The workout is drawn as a single ring near the watch bezel, broken into segments
proportional to each step's duration — red for HEAT, blue for REST, gray for an
optional COOLDOWN — starting at the top (12 o'clock) and running clockwise. A small
white notch/triangle marks the 12 o'clock start. The center holds the headline.

The big win: the **same ring is reused live during the workout** as the progress
indicator, so "here's your workout" and "here's where you are" share one visual.

## Where it's used
- **Suggestion screen 1** (was the bar chart) — preview ring + center headline.
- **Custom preview** (`CustomView` selection) — same preview ring.
- **Live HEAT/REST screens during a structured workout** — same ring, now showing
  progress (see Live mode).
- **NOT in free mode** — free sessions have no known durations, so the existing
  count-up screen stays as-is (no ring).

## Geometry (resolution-relative)
```
cx = w/2;  cy = h/2
ringR = (min(w,h) * 0.40)        // radius to the ring centerline
penW  = (min(w,h) * 0.075)       // ring thickness
totalSec  = sum of every step.durationSec (heat + rest + cooldown)
degPerSec = 360.0 / totalSec
```

## Drawing the segments (Monkey C)
Use `dc.setPenWidth(penW)` + `dc.drawArc(cx, cy, ringR, Graphics.ARC_CLOCKWISE,
startDeg, endDeg)` per segment. Garmin's drawArc uses degrees measured
counter-clockwise from 3 o'clock (0=E, 90=N/top), so start at 90 and sweep clockwise
by decreasing the angle:
```
var cursor = 90.0;                       // top
for each step in workout.steps {
    var span  = step.durationSec * degPerSec;
    var end   = cursor - span;           // clockwise
    dc.setColor(colorFor(step.type), Graphics.COLOR_TRANSPARENT);
    dc.drawArc(cx, cy, ringR, Graphics.ARC_CLOCKWISE, cursor, end);
    cursor = end;
}
```
- `colorFor`: HEAT → `0xE0392C`, REST → `0x3E8FD0`, COOLDOWN → `0x7d8088`.
- Draw the start notch: a small white triangle at `(cx, cy - ringR - penW/2 - 3)`.
- **Verify sweep direction in the simulator once** — if it runs counter-clockwise,
  swap to `Graphics.ARC_COUNTER_CLOCKWISE` or negate the span. (drawArc orientation is
  the one thing worth eyeballing.)
- Tiny segments: enforce a minimum visible span (e.g. clamp each segment to ≥ ~4°) so
  a very short rest still shows.

## Preview center content
- Line 1 (large): "N × H" (rounds × heat minutes), e.g. "3 × 12".
- Line 2 (muted): "min in heat".
- Line 3 (muted): total minutes (e.g. "36 min total").
- For the Race kind, the kind badge / "Race prep" can sit just under the headline.

## Live mode (during a structured workout)
Reuse the ring on the HEAT/REST screens instead of a bare countdown:
- Draw all segments at full color so the whole shape stays visible.
- Compute progress: `elapsedInWorkout = (sum of completed steps' durationSec) +
  _phaseElapsed`; `progressDeg = elapsedInWorkout * degPerSec`.
- Draw a **white position marker** (a short radial tick or a filled dot) on the ring at
  angle `90 - progressDeg`. Optionally dim already-completed segments.
- Center shows: the **current block's remaining time counting DOWN**
  (`step.durationSec - _phaseElapsed`), "Round N of M", and a one-line next-block
  preview — same data as BUILD_TODO §12, just framed by the ring.
- On auto-advance or a LAP skip, the marker jumps to the new segment automatically
  (it's derived from elapsed time + `_stepIndex`, so no extra wiring).
- **Over-run:** once past the final step, the workout is complete — show the ring full
  and fall through to the free count-up screen per BUILD_TODO §12.

## Edge cases
- A step with `durationSec == null` (manual) has no proportional span — structured
  workouts shouldn't contain manual steps, but if one appears, give it a small fixed
  span and advance only on LAP.
- One-round / very short workouts still render (the ring just has fewer segments).

## Acceptance criteria
- Suggestion screen 1 and the custom preview show the segmented ring (red heat / blue
  rest), a start notch at the top, and the centered headline — no bar chart.
- During a structured workout the same ring shows a moving position marker, the current
  block counts down in the center with "round N of M", and LAP/auto-advance moves the
  marker correctly.
- Free sessions are unaffected (no ring; existing count-up screen).
- All geometry is fractions of `min(w,h)` so it scales across devices.
