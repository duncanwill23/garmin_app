# Race date UI redo — Menu2 first, picker second (matches IdleMenu pattern)

Replaces the single cramped picker screen with the app's standard two-step pattern:
a `Menu2` that adapts to saved state, then the UP/DOWN picker. This removes the
button conflict (LAP-to-delete vs. how-to-go-back) entirely.

Key in `Config.mc` (define once, reference everywhere):
```monkeyc
const KEY_RACE_DATE = "race_date_epoch";
```
Use `Config.KEY_RACE_DATE` in RaceDateView and HeatAccDelegate so writer/reader match.

---

## Flow

```
Idle options menu  ──(Set race date)──►  RaceDateMenu (Menu2)
                                          │
   if NO date saved:                      ├─ "Set race date"   ──► RaceDatePicker
                                          └─ (BACK → idle options)
                                          │
   if date saved (title = "Race in Nd"):  ├─ "Change date"     ──► RaceDatePicker
                                          ├─ "Clear race date" ──► delete + back to idle
                                          └─ (BACK → idle options)
```
Picker buttons: UP/DOWN adjust, START save+return to idle, BACK cancel→RaceDateMenu.
No LAP overload anywhere.

---

## 1. New `RaceDateMenu` (Menu2) + delegate

Add to `source/RaceDateView.mc` (or a new file). Mirrors `IdleMenu`/`IdleMenuDelegate`.

```monkeyc
class RaceDateMenu extends WatchUi.Menu2 {
    function initialize(view) {
        var saved = Storage.getValue(Config.KEY_RACE_DATE);
        var days  = null;
        if (saved != null) {
            var d = (saved - TrendStore.nowEpoch()) / 86400;
            if (d > 0) { days = d; }
        }

        var title = (days != null)
            ? "Race in " + days + "d"
            : WatchUi.loadResource(Rez.Strings.MenuRaceDate);
        Menu2.initialize({ :title => title });

        if (days != null) {
            addItem(new WatchUi.MenuItem(
                WatchUi.loadResource(Rez.Strings.RaceChange), null, :change, null));
            addItem(new WatchUi.MenuItem(
                WatchUi.loadResource(Rez.Strings.RaceClear), null, :clear, null));
        } else {
            addItem(new WatchUi.MenuItem(
                WatchUi.loadResource(Rez.Strings.RaceSet), null, :change, null));
        }
    }
}

class RaceDateMenuDelegate extends WatchUi.Menu2InputDelegate {
    private var _mainView;
    function initialize(mainView) {
        Menu2InputDelegate.initialize();
        _mainView = mainView;
    }
    function onTap(evt)   { return true; }
    function onHold(evt)  { return true; }
    function onSwipe(evt) { return true; }

    function onSelect(item) {
        var id = item.getId();
        if (id == :change) {
            WatchUi.switchToView(new RaceDatePicker(_mainView),
                new RaceDatePickerDelegate(_mainView), WatchUi.SLIDE_LEFT);
        } else if (id == :clear) {
            Storage.deleteValue(Config.KEY_RACE_DATE);
            // back to idle options menu
            WatchUi.switchToView(new IdleMenu(_mainView),
                new IdleMenuDelegate(_mainView), WatchUi.SLIDE_RIGHT);
        }
    }
    function onBack() {
        WatchUi.switchToView(new IdleMenu(_mainView),
            new IdleMenuDelegate(_mainView), WatchUi.SLIDE_RIGHT);
        return true;
    }
}
```

New strings in `strings.xml`:
```xml
<string id="RaceSet">Set race date</string>
<string id="RaceChange">Change date</string>
<string id="RaceClear">Clear race date</string>
```

## 2. Rename the existing view to `RaceDatePicker` and simplify it

The current `RaceDateView` becomes `RaceDatePicker` — same UP/DOWN/START picker, but:
- Remove the LAP/clear responsibility entirely (clear now lives in the menu).
- Bottom help line becomes just: `"UP/DOWN adjust · START save"`.
- `save()` unchanged (writes `Config.KEY_RACE_DATE`).
- `initialize` still preloads the saved days so "Change date" opens on the current value.

Picker delegate (`RaceDatePickerDelegate`):
```monkeyc
function onTap(evt)   { return true; }
function onHold(evt)  { return true; }
function onSwipe(evt) { return true; }
function onNextPage()     { /* decrement */ ... return true; }
function onPreviousPage()  { /* increment */ ... return true; }
function onKey(keyEvent) {
    var key = keyEvent.getKey();
    if (key == WatchUi.KEY_ENTER || key == WatchUi.KEY_START) {
        var v = WatchUi.getCurrentView()[0];
        if (v instanceof RaceDatePicker) { v.save(); }
        // return to idle in one step
        WatchUi.switchToView(_mainView, new HeatAccDelegate(_mainView), WatchUi.SLIDE_RIGHT);
        return true;
    }
    return false;
}
function onBack() {
    // cancel — back to the race menu, not idle
    WatchUi.switchToView(new RaceDateMenu(_mainView),
        new RaceDateMenuDelegate(_mainView), WatchUi.SLIDE_RIGHT);
    return true;
}
```

## Acceptance criteria
- Selecting "Set race date" with no race saved shows a one-item menu → picker.
- Selecting it with a race saved shows a menu titled "Race in Nd" with "Change date"
  and "Clear race date".
- "Clear race date" deletes the date and the race suggestion stops appearing.
- Picker uses UP/DOWN to adjust, START to save+return to idle, BACK to return to the
  race menu. LAP does nothing on the picker.
- BACK from the race menu returns to the idle options menu, consistent with every
  other sub-menu.
- The saved date persists across app open/close and watch restart.
