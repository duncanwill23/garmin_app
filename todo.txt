# Fix: block touch input everywhere except suggestion scroll screen

## The problem
On the fr955, `BehaviorDelegate.onSelect()` is triggered by both the physical
START/Enter button AND a tap anywhere on the touchscreen — Garmin maps them to the
same behavior internally. Removing `onTap` does nothing because the tap is translated
into `onSelect` before the app ever sees it as a touch event.

## The fix
Intercept touch at the `InputDelegate` level by implementing `onTap`, `onHold`, and
`onSwipe` and returning `true` (consumed) in every delegate except the suggestion
explanation screen. Returning `true` signals the input is handled, which prevents
Garmin from translating it into the corresponding behavior (`onSelect`, `onNextPage`,
`onPreviousPage`). Physical buttons are unaffected.

## Add these three methods to EVERY delegate in the app

```monkeyc
function onTap(clickEvent as WatchUi.ClickEvent) as Lang.Boolean {
    return true;
}
function onHold(clickEvent as WatchUi.ClickEvent) as Lang.Boolean {
    return true;
}
function onSwipe(swipeEvent as WatchUi.SwipeEvent) as Lang.Boolean {
    return true;
}
```

Apply to all of these files:
- `HeatAccDelegate.mc` — all delegate classes inside it
- `StopMenuDelegate.mc`
- `IdleMenuDelegate.mc` (inside `HeatAccDelegate.mc` or its own file)
- `SuggestionView.mc` — the chooser delegate and screens 1 and 3's delegate
- `CustomView.mc` — all delegates
- `RaceDateView.mc` — all delegates
- `RestDayView.mc`
- `SummaryView.mc` delegate
- `AcclimationView.mc` delegate
- `SessionGraphView.mc` delegate
- Any other delegate class in the project

Do NOT remove or change `onSelect` — it must stay for the physical START/Enter button
to keep working. These three methods only block the touch path.

## The one exception — suggestion screen 2 (explanation scroll view)
The delegate for `SuggestionView` screen 2 (the scrollable explanation text) should
NOT implement `onSwipe`, so the user can still scroll the text with a finger swipe.
It SHOULD still implement `onTap` and `onHold` returning `true` to block accidental
taps. So for that delegate only:

```monkeyc
function onTap(clickEvent as WatchUi.ClickEvent) as Lang.Boolean {
    return true;
}
function onHold(clickEvent as WatchUi.ClickEvent) as Lang.Boolean {
    return true;
}
// onSwipe intentionally omitted — allows scroll
```

## Acceptance criteria
- Tapping anywhere on any screen (idle, heat, rest, summary, menus, race date,
  rest day) does nothing — no session starts, no pause, no navigation.
- The physical START, LAP, BACK, UP, DOWN buttons all still work normally.
- Swiping on suggestion screen 2 still scrolls the explanation text.
- Holding the screen does nothing on any screen.
